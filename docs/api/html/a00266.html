boost::filesystem::path boost::dll::program_location ( boost::system::error_code &  ec)
inline

On success returns full path and name of the currently running program (the one which contains the main() function).

Return value can be used as a parameter for shared_library. See Tutorial "Linking plugin into the executable" for usage example. Flag '-rdynamic' must be used when linking the plugin into the executable on Linux OS.

Parameters
ecVariable that will be set to the result of the operation.
Exceptions
std::bad_allocin case of insufficient memory. Overload that does not accept boost::system::error_code also throws boost::system::system_error.

template<class T >
boost::filesystem::path boost::dll::symbol_location ( const T &  symbol,
boost::system::error_code &  ec

inline

On success returns full path and name of the binary object that holds symbol.

Template Parameters
TType of the symbol, must not be explicitly specified.
Parameters
symbolSymbol which location is to be determined.
ecVariable that will be set to the result of the operation.
Returns
Path to the binary object that holds symbol or empty path in case error.
Exceptions
std::bad_allocin case of insufficient memory. Overload that does not accept boost::system::error_code also throws boost::system::system_error.

Examples:

int var;
void foo() {}
int main() {
dll::symbol_location(var); // returns program location
dll::symbol_location(foo); // returns program location
dll::symbol_location(std::cerr); // returns location of libstdc++: "/usr/lib/x86_64-linux-gnu/libstdc++.so.6"
dll::symbol_location(std::placeholders::_1); // returns location of libstdc++: "/usr/lib/x86_64-linux-gnu/libstdc++.so.6"
dll::symbol_location(std::puts); // returns location of libc: "/lib/x86_64-linux-gnu/libc.so.6"
}

Parameters
ptr_to_symbolPointer to symbol which location is to be determined.
ecVariable that will be set to the result of the operation.
Returns
Path to the binary object that holds symbol or empty path in case error.
Exceptions
std::bad_allocin case of insufficient memory. Overload that does not accept boost::system::error_code also throws boost::system::system_error.

Examples:

int main() {
dll::symbol_location_ptr(std::set_terminate(0)); // returns "/some/path/libmy_terminate_handler.so"
dll::symbol_location_ptr(::signal(SIGSEGV, SIG_DFL)); // returns "/some/path/libmy_symbol_handler.so"
}
